PUBLIC repo: glintt-next/.github
================================

This is the PUBLIC organization repository. It must be public so the profile
README renders as the landing page at github.com/glintt-next.

It contains ONLY:
  - profile/README.md  (branding + links into the internal docs repo)
  - profile/assets/     (logo + banner SVG)
  - PULL_REQUEST_TEMPLATE.md and ISSUE_TEMPLATE/  (auto-apply org-wide)

No sensitive content lives here. The client list, registry, guidelines, and
onboarding live in the INTERNAL repo glintt-next/data-ai-docs.

Set up
------
1. Create a PUBLIC repo in the org named exactly:  .github
2. Push these contents to main:
     git init && git add . && git commit -m "chore: org profile and templates"
     git branch -M main
     git remote add origin git@github.com:glintt-next/.github.git
     git push -u origin main

The README links to https://github.com/glintt-next/data-ai-docs/... . Those
links are public text, but the destination repo is internal: enterprise members
resolve them; everyone else gets a 404. Create data-ai-docs (see its own SETUP)
so the links work for members.

If the banner SVG does not render over the raw URL, switch the <img src> to the
relative path profile/assets/glintt-next-dataai-banner.svg or commit a PNG.
